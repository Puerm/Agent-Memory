Memory 不是“把历史对话塞进向量库”，而是 Agent 对过去经历进行选择、压缩、校验、调用和遗忘的决策系统。

长期记忆的核心难题不是 Recall，而是 **Admission**：什么信息有资格影响未来的 Agent？

事件不是经验；被提炼为未来行动策略的事件，才是经验。


时间不是 metadata，而是 Memory 的一等公民。
Graphiti 会为事实保留有效期和来源，并在新事实冲突时关闭旧事实的有效期，而不是粗暴删除。

数据库保存过去；Memory 管理“过去如何有资格影响现在”。


如果只是将整个旧文本归档再生成新文本，就可能丢失那些“用户没有重新说、但仍然有效”的信息。这叫做 **silent information loss（静默信息丢失）**。

向量检索回答“这两段话像不像”；Memory Retrieval 回答“这条过去的信息现在是否有资格影响我的行动”。

当前指令可以暂时覆盖记忆，但不一定有权改写记忆。
例如，Memory 记录“用户偏好中文”，但用户说：

> “这次给外国同事生成英文版。”

本次应使用英文，但不能把长期偏好改成英文。因为这是局部任务要求，不是全局偏好更新。

这类机制可以称为 **Working-memory overlay**：



Agent Memory 不是让模型记得更多，而是让过去的信息正确地影响现在的行动。


一条记忆的价值，等于移除它之后 Agent 行为发生了多大变化。
